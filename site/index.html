Quick Start

Install a Nebari Helm chart

Choose your preferred installation method below.

1

Add the Helm repository

Register the Nebari Helm repository under the alias nebari.

helm repo add nebari \ https://raw.githubusercontent.com/nebari-dev/helm-repository/gh-pages/
2

Update the local cache

Pull the latest chart index from the repository.

helm repo update nebari
3

Install a chart

Browse the charts below, then install the one you need.

# Replace <chart-name> with the desired chart helm install my-release nebari/<chart-name>

Private repository access

If you are using a private fork of this repository (requires a GitHub Enterprise license), authenticate with a personal access token (PAT):

helm repo add nebari \ https://raw.githubusercontent.com/nebari-dev/helm-repository/gh-pages/ \ --username <your-github-username> \ --password <your-PAT>
1

Install directly — no helm repo add needed

Charts are published to quay.io/nebari-dev as public OCI artifacts. Use the chart's version tag directly.

# --version is required for OCI refs helm install my-release \ oci://quay.io/nebari-dev/<chart-name> \ --version <version>
2

Pull or inspect without installing

# Download the chart archive helm pull oci://quay.io/nebari-dev/<chart-name> --version <version> # Inspect chart metadata helm show chart oci://quay.io/nebari-dev/<chart-name> --version <version>

No authentication required

All Nebari OCI charts on quay.io are public. You do not need to run helm registry login to pull or install them. The version tag for each chart is shown in the chart catalog below.

About

How it works

This repository is powered by GitHub Actions and GitHub Pages — no external infrastructure required.

📦

Automated publishing

Push a packaged .tgz chart to the charts/ folder on main and a GitHub Actions workflow will update the index and deploy it automatically.

🔒

Access control

Public repositories are accessible to everyone. Private forks (GitHub Enterprise) support token-based authentication via the raw.githubusercontent.com endpoint.

🏷️

Versioned index

Each chart version is preserved in index.yaml with its original creation date, so helm repo update always gives you the full history.

�

OCI on quay.io

Every chart is also pushed to quay.io/nebari-dev as a public OCI artifact, matching Nebari's existing container image workflow. Install with a single helm install oci://… command — no repo registration needed.

🚀

Zero maintenance

No ChartMuseum, no external object storage. Everything is automated via GitHub Actions — GitHub Pages serves the Helm index and quay.io serves the OCI artifacts.
